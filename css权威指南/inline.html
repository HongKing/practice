行内元素

非替换元素的内边距、边框和外边距对行内元素或其生成的框没有垂直的效果，即不会影响元素行内框的高度，行内框在行中根据其vertical-align属性值垂直对齐

line-height实际只影响行内元素和其他行内内容，而不影响块级元素，至少不会直接影响块级元素，只会影响块级元素的内联内容

行内替换元素

vertical-align:bottom;把行内框的的底端与行框底端对齐，vertical-align:top;则把行内框的顶端与行框的顶框对齐,line-height值则可增加会减少行内框的值，对应可以提高或减少行框的值

line-height可以提高行框的总高度，从而提供足够的空间来显示元素，而不会与其他文本重叠
记住，line-height相对与元素本身的font-size设置，而不是相对于父元素设置

行内元素的边框边界由font-size而不是line-height控制，如果要把边框从文本拉开，则设置内边距;
如果内边距足够大，则会重叠到其他行上面
外边距不会应用到行内非替换元素的顶端和底端

line-height属性的细节

语法：line-height: normal | - number | - length | - percentage

- normal 根据浏览器决定，一般为1.2。
- number 仅指定数字时（无单位），实际行距为字号乘以该数字得出的结果。可以理解为一个系数，子元素仅继承该系数，子元素的真正行距是分别与自身元素字号相乘的计算结果。大多数情况下推荐使用，可以避免一些意外的继承问题。
- length 具体的长度，如px/em等。
- percentage 百分比，100%与1em相同。
有单位（包括百分比）与无单位之间的区别 有单位时，子元素继承了父元素计算得出的行距；无单位时继承了系数，子元素会分别计算各自行距（推荐使用）
