行内元素

行内替换元素指的是button，input，img 等

行内非替换元素有span，strong，em，i，time 等

2. 行内元素特性

width, height 无效

padding 上下左右都有效，但上下内边距无法撑开行内框以及父级元素，但会撑开自身的边框边界，同时也导致行内元素之间上下重叠的问题

margin 左右有效，但上下外边距无效

边框 行内元素的边框边界由font-size而不是line-height控制，如果要收缩扩展文本边框，则设置内边距;

行内框高度 由line-height设置，内边距、边框和外边距对行内元素或其生成的框没有垂直的效果，即不会影响元素行内框的高度

如下的 padding无法撑开父级导致溢出，无法撑开自身行内框导致上下行内元素重叠

margin 有左右外边距，但没有上下外边距

padding:20px margin:20px
padding:20px margin:20px

如下的 line-height 撑开行内框，行内框进而撑开父级元素，溢出父级的部分就是边框边界大于行内框的部分，padding扩展出的边框边界大于行内框，所以上下元素仍然会重叠

line-height:40px;padding: 20px;
margin:20px;padding: 10px;

3. vertical-align 设置行内框内垂直对齐

vertical-align:bottom; 把行内框的的底端与行框底端对齐，
vertical-align:top; 则把行内框的顶端与行框的顶框对齐

vertical-align:bottom; this is text vertical-align:top;

4. line-height 设置行内框的高度

line-height 只影响行内元素和其他行内内容，而不影响块级元素，至少不会直接影响块级元素，只会影响块级元素的内联内容
line-height 设置行框的高度，从而提供足够的空间来显示元素，而不会与其他文本重叠
line-height 相对与元素本身的font-size设置，而不是相对于父元素设置

this is text some of which is emphasized ，plus other text is strongly emphasized and which is tall and larger than the surrounding

5. line-height属性的细节

语法：line-height: normal | - number | - length | - percentage

- normal 根据浏览器决定，一般为1.2

- number 仅指定数字时（无单位），实际行距为字号乘以该数字得出的结果。可以理解为一个系数，子元素仅继承该系数，子元素的真正行距是分别与自身元素字号相乘的计算结果。大多数情况下推荐使用，可以避免一些意外的继承问题

- length 具体的长度，如px/em等。

- percentage 百分比，100%与1em相同。

有单位（包括百分比）与无单位之间的区别

有单位时，子元素继承了父元素计算得出的行距；无单位时继承了系数，子元素会分别计算各自行距（推荐使用）
